# 角色生成提示词规范 v1.0

> 标准化、可扩展、平台自适应的角色图像生成提示词体系
> 版本: 1.0 | 日期: 2026-06-09 | 维护: AI 剧本团队

---

## 1. 体系总览

本规范定义了**5 画风 × 3 模板 × 7 平台 × 4 层架构**的角色生成提示词体系,
解决以下痛点:
- 不同画风输出的角色形象不一致 (缺乏风格锚点)
- 平台参数混乱 (MJ/SD/Flux 混用参数)
- 配角/路人描述单薄, 主角描述冗余
- 新增画风/平台需要修改多处代码

### 1.1 核心组件

| 组件 | 文件 | 数量 | 职责 |
|------|------|------|------|
| 画风定义 | `styles/*.json` | 5 | 画风关键词/光影/相机/胶片/负面词 |
| 模板级别 | `templates/*.json` | 3 | basic/advanced/professional 三档详细度 |
| 平台参数 | `templates/*.json#L4_platform` | 7 | MJ/SD/Flux/DALL-E/Agnes 参数适配 |
| 角色检测 | `builders/role-detector.ts` | - | 主角/配角/路人自动识别 |
| Prompt 构建 | `builders/prompt-builder.ts` | - | 读取配置 → 输出完整 prompt |
| 预设 | `presets/*.json` | 75 | 5 角色 × 5 画风 × 3 模板 |

## 2. 四层架构 (4-Layer Architecture)

### 2.1 L1 - 核心主体层 (Highest Priority)

**职责**: 角色核心身份信息, 不可被任何后续层覆盖

| 字段 | 必填 | 说明 |
|------|------|------|
| `character_name` | ✓ | 角色名 |
| `archetype` | ✓ | 角色原型 (tyrant emperor / wuxia hero / ...) |
| `role` | ✓ | 角色功能 (protagonist / supporting / background) |
| `gender` | ✓ | 性别 |
| `age` | ✓ | 年龄 |
| `physical_traits` | ✓ | 物理特征 (脸型/眼/发/肤/体态) |
| `appearance` | ✓ | 外貌 (服装/配饰/道具) |
| `personality` | ✗ | 性格 + 默认表情 + 情绪范围 |
| `background` | ✗ | 出身 + 人际关系 |
| `signature` | ✗ | 显著特征 + 标志动作 + 标志物 |

**合并策略**: `override_protected` (最高优先级, 任何层不可覆盖)

### 2.2 L2 - 风格氛围层

**职责**: 定义视觉风格、色彩、氛围, 作为绘制指导

| 字段 | 必填 | 示例 |
|------|------|------|
| `style_keywords` | ✓ | 风格核心关键词 (来自 styles/*.json) |
| `color_palette` | ✗ | teal/orange, monochrome, neon, vibrant |
| `mood` | ✗ | melancholic, epic, ethereal, gritty, menacing |
| `era` | ✗ | ancient china, cyberpunk 2077, tang dynasty |
| `composition` | ✗ | rule of thirds, golden ratio, centered |
| `artistic_influences` | ✗ | "like Wong Kar-wai", "Ghibli-esque" |

**合并策略**: `overlay` (叠加, 不覆盖 L1)

### 2.3 L3 - 技术参数层

**职责**: 渲染技术细节, 决定输出质量

| 子层 | 字段 | 说明 |
|------|------|------|
| camera | lens/focal_length/aperture/sensor/angle/movement | 镜头参数 |
| lighting | type/direction/quality/color_temp/key/fill/rim | 布光 |
| rendering | engine/resolution/samples/denoiser/GI/caustics/SSS | 渲染引擎 |
| materials | skin/hair/cloth/metal/glass/fabric | 材质 |
| postProcessing | color_grading/grain/bloom/vignette/CA | 后期 |

**合并策略**: `append` (追加, 不冲突)

### 2.4 L4 - 平台参数层

**职责**: 平台特定参数, 运行时根据目标平台注入

| 平台 | 关键参数 |
|------|----------|
| **Midjourney** | `--ar 3:4 --v 6.1 --s 250 --style raw --cref {url} --cw 100` |
| **StableDiffusion** | `--steps 30 --cfg 7 --sampler dpmpp_2m_karras` |
| **Flux-dev** | `--guidance 3.5 --steps 50 --width 1024 --height 1536` |
| **Flux-pro** | `--guidance 4.0 --steps 28 --width 1024 --height 1536` |
| **DALL-E 3** | `--quality hd --size 1024x1792 --style vivid` |
| **Agnes Image** | `image_url: {ref_url}` (text prompt + multimodal) |

**合并策略**: `platform_specific` (运行时切换)

## 3. 五种画风 (5 Styles)

### 3.1 超写实真人电影画风 (`hyperrealistic-cinematic`)

**关键词**: hyperrealistic, photorealistic, 8k, 16k, ultra-detailed, skin texture, pores, subsurface scattering, peach fuzz, imperfections, micro details, micro wrinkles, freckles

**镜头**: 85mm portrait lens, f/1.4, f/1.8, 135mm portrait, shallow depth of field, bokeh, 85mm portrait lens, full frame, ARRI Alexa 65, Sony A7R IV, ARRI Alexa LF, IMAX, ARRI Alexa Mini LF, Zeiss Supreme Primes

**布光**: cinematic lighting, Rembrandt lighting, rim light, volumetric lighting, volumetric fog, god rays, chiaroscuro, catchlight

**胶片**: Kodak Portra 400, Kodak Vision3 500T, Fujifilm Pro 400H, Kodak Vision3 250D, Kodak Ektachrome, Cinestill 800T

**调色**: teal and orange, cinematic color grading, LUT, DaVinci Resolve grade, film grain

**适用场景**: 现代剧/古言/武侠 (追求真实质感)
**负面**: cartoon, illustration, painting, drawing, anime, 3d render, cgi, blurry, low quality, distorted, deformed, ugly, bad anatomy, extra limbs, missing fingers, watermark, text, signature

### 3.2 水墨古风 (`chinese-ink-wash`)

**关键词**: Chinese ink wash, gongbi fine brushwork, xieyi freehand, rice paper, mineral pigments, five tones of ink, liubai negative space

**笔法**: 工笔 (gongbi) / 写意 (xieyi) 二选一或混用

**适用场景**: 古风/仙侠/历史正剧
**特色**: 留白美学, 不强求背景细节

### 3.3 赛博朋克 (`cyberpunk`)

**关键词**: cyberpunk 2077, neon noir, tech-noir, rain-soaked streets, holographic UI, chromatic aberration, neon pink/cyan, blade runner, dystopian future

**适用场景**: 科幻/反乌托邦/未来
**特色**: 霓虹 + 雨夜 + 义体

### 3.4 国漫动漫风 (`guoman-anime`)

**关键词**: Chinese animation, donghua, cel shading, clean linework, vibrant flat colors, large expressive eyes, Fog Hill of Five Gods, White Cat Legend

**适用场景**: 国漫风/IP 改编/年轻向
**特色**: 中国动画学派 (大鱼海棠/罗小黑/白蛇等风格)

### 3.5 3D CG 动画风 (`3d-cg`)

**关键词**: Pixar-quality, octane render, subsurface scattering, stylized realism, Disney/Pixar/DreamWorks, path tracing, physically based rendering

**适用场景**: 儿童向/合家欢/Q 版
**特色**: 迪士尼/皮克斯/梦工厂 3D 渲染质感

## 4. 三级模板 (3 Templates)

### 4.1 基础版 (`basic`)

- **目标用户**: 新手, 快速生成
- **字段数**: 5 (核心)
- **输出长度**: ~100-200 字符
- **包含层**: L1 + L2 + L3(简化) + L4

### 4.2 进阶版 (`advanced`) ⭐ 推荐

- **目标用户**: 常规生成
- **字段数**: 10
- **输出长度**: ~600-1000 字符
- **包含层**: L1 + L2 + L3 + L4 (含 5 平台预设)

### 4.3 专业版 (`professional`)

- **目标用户**: 美术/导演, 精细控制
- **字段数**: 20+ (四层全展开)
- **输出长度**: ~1000-2000 字符
- **包含层**: L1 + L2 + L3 + L4 + 上下文感知 + 平台自适应

## 5. 角色自动检测 (Role Auto-Detection)

### 5.1 信号模型

| 信号 | 权重 | 主角阈值 |
|------|------|----------|
| descriptionLength | 0.4 (>=500) | 500+ 字符 |
| mentionCount | 0.3 (>=15) | 15+ 次 |
| hasCoreConflict | 0.3 | 匹配关键词 |
| hasExplicitProtagonistTag | 0.5 | "主角/hero/MC" |
| hasUniquePronoun | 0.15 | 独立指代 |
| referencedByOthers | 0.2 (>=5) | 被 5+ 角色引用 |

### 5.2 核心冲突关键词

中英双语覆盖: 复仇, revenge, 爱情, love, 仇恨, hatred, 责任, duty, 救赎, redemption, 成长, growth, 使命, mission, 命运, destiny, 暴君, tyrant, 背叛, betrayal, 牺牲, sacrifice, 守护, protect, 灭门, 篡位, coup, 逃离, escape

### 5.3 判定规则

```
protagonistScore >= 0.5  → 主角 (boostFactor = 1.5)
protagonistScore + supportingScore >= 0.4  → 配角 (boostFactor = 1.2)
其他  → 路人 (boostFactor = 0.8)
```

**主角获得 +50% 详细度加成, 路人会被截断到 30% 描述长度**

## 6. 使用指南

### 6.1 服务端调用示例

```typescript
import { buildCharacterPrompt, listStyles, listTemplates } from '@ai-video-script/prompt-builder';

const result = buildCharacterPrompt({
  character: {
    id: 'char-1',
    name: '独孤琰',
    description: '# 基本信息\n- 年龄: 十八岁...',
    extraDescription: '# 性格\n- 冷峻阴鸷...',
    roleType: 'protagonist', // 可选, 不传则自动检测
  },
  styleId: 'hyperrealistic-cinematic',
  templateId: 'advanced',
  platform: 'midjourney',
  referenceImageUrl: 'https://cdn.example.com/ref.png',
  context: { era: 'ancient china', genre: 'ancient_romance' },
});

console.log(result.prompt);
console.log('Negative:', result.negativePrompt);
console.log('Metadata:', result.metadata);
```

### 6.2 前端调用示例

```typescript
// 调用后端 API
const response = await fetch('/api/characters/${id}/generate-images', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    style: 'hyperrealistic-cinematic',
    template: 'advanced',
    platform: 'midjourney',
    referenceImageUrl: 'https://...',
  }),
});
```

### 6.3 切换画风

用户在前端选择"水墨古风" → 后端调用 `buildCharacterPrompt({ styleId: 'chinese-ink-wash', ... })` → 返回新 prompt → 调 Agnes/MJ/SD 出图

## 7. 质量控制

### 7.1 闸门 (Quality Gates)

- **最小 prompt 长度**: 50 字符 (低于则警告)
- **最大 prompt 长度**: 2000 字符 (超长则截断, MJ 平台除外)
- **必填层**: L1 + L2 (缺一警告)
- **必填字段**: character_name, role, gender, age, physical_traits

### 7.2 负面词覆盖

- 通用负面词 (低质量/变形/水印) 自动从 style + template 合并去重
- 风格特异负面词 (hyperrealistic 排除 anime, ink-wash 排除 photo, ...) 自动注入

### 7.3 平台适配

- **Agnes Image**: 只输出 text prompt + image_url, 不带任何 `--flag` 参数
- **Midjourney**: 转 `--flag value` 形式, 自动去重
- **StableDiffusion**: 输出 `key=value` 形式
- **Flux/DALL-E**: 按官方文档格式

## 8. 扩展指南

### 8.1 新增画风

1. 在 `config/prompts/styles/` 创建 `your-style.json`:
```json
{
  "styleId": "your-style",
  "name": "你的画风",
  "description": "...",
  "category": "...",
  "keywords": { "subject": [...], "camera": [...], "lighting": [...], "film": [...], "colorGrading": [...], "negative": [...] },
  "negativePrompt": "..."
}
```

2. 测试: `tsx test-prompt-builder.ts` (应自动发现新画风)

3. 添加预设: `tsx preset-generator.ts`

### 8.2 新增模板

1. 在 `config/prompts/templates/` 创建 `your-template.json`
2. 必填字段: `templateId`, `name`, `description`, `level`, `layers.L1_core.template`, `promptTemplate`
3. 测试 + 预设

### 8.3 新增平台

1. 修改 `builders/prompt-builder.ts`:
   - `PlatformId` 联合类型加新值
   - `listPlatforms()` 数组加新值
2. 在 `templates/*.json#L4_platform.platformPresets` 添加新平台参数
3. 编写平台适配器 (若需要格式转换)

## 10. 已知限制

1. **角色检测基于启发式**, 不调用 LLM. 准确率 ~80%. 复杂场景可显式传 `roleType`
2. **平台适配器简化**: MJ/SD/Flux 参数格式略有差异, 当前按通用约定输出
3. **多语言**: 当前描述字段只支持中文. 若需英文, 需在 style 中添加 en_keywords
4. **画风关键词去重**: 部分 style JSON 有重复键 (如 `camera` 出现 2 次), 已被 prompt-builder 去重
