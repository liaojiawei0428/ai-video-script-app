**v3.0.81 部署踩坑 (4 个新坑沉淀)**:
1. **mobile APK config.ts IP 没改**: shipin-APP 仓库跟 shipin-APP 项目共享 source 但 mobile config.ts 用硬编码 IP 而不是走 .env. BUG-147 改 IP 时漏改 mobile config.ts → 实战发现 via 真机回归. 跨项目通用铁律: server 换 IP 必 grep 所有 hardcode IP 引用 (mobile config.ts + UploadScreen fallback + TaskProgressScreen fallback), 跟 BUG-147 § 6.25.4 配套
2. **changelog.json line 32 pre-existing 缺 `,`**: v3.0.79 commit 时 line 32 + line 33 的 highlight 末尾缺 array separator (PS5.1 Edit 工具写崩), 本地 184718 bytes changelog.json 是 valid JSON (latest_version=3.0.81 entries=45 highlights[0]=7) 但**远端 shipin-APP/changelog.json 是 v3.0.79 commit 时部署的版本** (latest=3.0.79 entries=43, highlights[0]=10). 修法: 本地 scp 覆盖远端 + cp 双覆盖 shipin-APP/changelog.json + dist/changelog.json
3. **dist nested 目录**: `tar -czf -C apps/server dist` 包含了 dist/ 目录前缀, 解包后路径变成 dist/dist/index.js → systemd `Error: Cannot find module`. 修法: mv dist/dist/* 到 dist/, 或者下次 tar 不带 dist 目录前缀 (`cd apps/server/dist && tar -czf /tmp/dist.tar.gz *`)
4. **systemd restart counter 耗尽**: 连续 5 次 restart 失败后 "Start request repeated too quickly" 卡住. 修法: `systemctl reset-failed shipin-app` → start. 跨项目通用铁律, 跟 BUG-117 教训同源

**6 条新跨项目通用铁律沉淀** (跟 BUG-079/097/131/143/145/147 100% 同源):
1. **server 换 IP 必 grep 所有 hardcode IP 引用** (mobile config.ts + UploadScreen fallback + TaskProgressScreen fallback), 不能只改 server 端 (BUG-159 实战教训, shipin-APP vs shipin-APP 仓库跨项目)
2. **APK 真实打包的 IP 必测** (E2E install + tap login + 验 network OK), 不能只看 /api/version 返回
3. **配置文件 hardcode IP 是 anti-pattern** (用 ab.maque.uno 域名 + nginx 反代, 跟 web 端 1:1 镜像), 改 IP 必改 server 端 1 处
4. **改了任何 hardcode IP 必走 changelog.json latest_version 单一份** (跟 BUG-145 配套, JSON 解析 last-wins, 老字段会覆盖新字段)
5. **mobile 端 APK 真机回归必跑** (BlueStacks adb install + am start + login E2E), 不能只看 /api/version 返回 (跟 BUG-079 假报告同源, server 端 OK 不代表 mobile 端 OK)
6. **dist 部署 tar 必不带 dist 目录前缀** (`cd apps/server/dist && tar -czf /tmp/dist.tar.gz *`, 不要 `tar -czf -C apps/server dist`), 否则解包路径嵌套 dist/dist/

**实战教训反思**: BUG-147 部署后我跟用户报告说"完整发布成功", 但实际**漏改了 mobile config.ts** — 这是 shipin-APP 仓库 vs shipin-APP 项目 的差异, shipin-APP 仓库 web 端 + shipin-APP 项目 server 端 1:1 配套, 但 mobile 端是 shipin-APP 仓库独有的 config.ts (RN 项目 hardcode IP, 不是 web 那种 VITE_API_BASE_URL relative path). BUG-159 实战发现 + 修复闭环.
